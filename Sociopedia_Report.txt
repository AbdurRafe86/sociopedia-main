ABSTRACT

Sociopedia is a full-featured social media platform designed to provide users with a secure and interactive environment for sharing content and connecting with friends. Built using the MERN stack (MongoDB, Express.js, React, and Node.js), the application emphasizes a premium user experience through a modern design system, dark/light mode functionality, and real-time social interactions. The system implements secure JWT-based authentication, image-based registration, and an optimized backend for efficient data retrieval. Key features include creating media-rich posts, a friend management system, and profile analytics such as profile views and impressions. This project demonstrates the practical application of modern web development technologies to build a scalable and responsive social networking solution.

CHAPTER-1
1.1 INTRODUCTION
Social media has become an integral part of modern communication, allowing individuals to share life updates, media, and thoughts instantly. Sociopedia is a social media application developed to bridge the gap between simple social tools and complex enterprise platforms. By leveraging the MERN stack, the project provides a robust architecture that handles high-concurrency requests while maintaining a smooth user interface.

1.2 OBJECTIVES
The primary objective of Sociopedia is to create a secure, responsive, and user-friendly social media platform. The project aims to:
- Implement a secure authentication system using JSON Web Tokens (JWT).
- Provide a seamless interface for users to share media and text posts.
- Develop a connectivity system where users can add or remove friends.
- Offer a customizable user experience with light and dark theme support.

1.3 PROJECT INTERFACE
The interface is built using React and Material UI, following a clean and minimalist design. It consists of a navigation bar, a side panel for user info and friend lists, a central feed for posts, and a post creation widget. The design is fully responsive, ensuring accessibility across various devices.

1.4 BACKGROUND
Conventional social media platforms often suffer from privacy concerns and cluttered interfaces. Sociopedia addresses these issues by providing a simplified yet feature-rich experience. The use of modern frameworks like React and Node.js ensures that the application is fast and scalable.

1.5 PROBLEM STATEMENT
Developing a social media application requires balancing security, performance, and user engagement. Existing solutions may lack efficient image handling or simple friend management systems for small to medium-scale communities. There is a need for a modular social platform that can be easily customized and extended.

1.6 AIMS AND OBJECTIVES
- To build a backend API that manages user data, posts, and social relationships.
- To design a frontend that provides intuitive navigation and real-time feedback.
- To ensure data integrity and security through encryption and token-based access control.

2. Navigate to the server directory and run npm install.

4. Navigate to the client directory and run npm install.
5. Start the server using npm start and the client using npm start.

CHAPTER-4
4.1 PROBLEM DESCRIPTION
The core challenge is to manage the complex relationships between users, posts, and interactions (likes/comments) in a way that remains performant as the data grows.

4.2 USER AUTHENTICATION AND PROFILE MODULE
This module handles user registration, login, and profile management. It uses bcrypt for password hashing and Multer for profile picture uploads. User data including location, occupation, and social links are stored in MongoDB.

4.3 POSTS AND SOCIAL INTERACTION MODULE
This module allows users to create posts with images and descriptions. It also manages the feed by fetching posts from the database and sorting them by date. The social interaction part handles friend requests and the post liking system.

CHAPTER-5
5.1 SOFTWARE REQUIREMENTS SPECIFICATION
The system must allow users to:
- Register with a unique email and secure password.
- Upload images as part of their profile or posts.
- View a feed of posts from all users.
- Add or remove other users from their friend list.
- Toggle between dark and light modes.

5.2 SOFTWARE DESIGN
The project follows a MVC (Model-View-Controller) pattern on the backend.
- Models: User.js and Post.js define the data structure.
- Controllers: Logic for auth, users, and posts.
- Routes: Express routes that map URLs to controllers.
The frontend uses a component-based architecture with Redux for state management.

6.1 PROBLEM DISCUSSION
One specific implementation hurdle was handling file uploads alongside JSON data in the registration phase. This was solved using Multer middleware which intercepts multipart/form-data requests.

6.2 IMPLEMENTATION OF PROBLEM
The backend is initialized with Express and connected to MongoDB using Mongoose. Security headers are added via Helmet, and Cross-Origin Resource Sharing is enabled through CORS. JWT tokens are verified in a custom middleware before granting access to protected routes like creating posts or fetching friend lists.

--- USER MODEL (User.js) ---
const UserSchema = new mongoose.Schema(
  {
    firstName: { type: String, required: true },
    lastName: { type: String, required: true },
    email: { type: String, required: true, unique: true },
    password: { type: String, required: true },
    picturePath: { type: String, default: "" },
    friends: { type: Array, default: [] },
    location: String,
    occupation: String,
  },
  { timestamps: true }
);

--- REGISTER CONTROLLER (auth.js) ---
export const register = async (req, res) => {
  try {
    const { firstName, lastName, email, password, picturePath, friends, location, occupation } = req.body;
    const salt = await bcrypt.genSalt();
    const passwordHash = await bcrypt.hash(password, salt);
    const newUser = new User({
      firstName, lastName, email, password: passwordHash, picturePath, friends, location, occupation,
      viewedProfile: Math.floor(Math.random() * 10000),
      impressions: Math.floor(Math.random() * 10000),
    });
    const savedUser = await newUser.save();
    res.status(201).json(savedUser);
  } catch (err) {
    res.status(500).json({ error: err.message });
  }
};

--- CREATE POST CONTROLLER (posts.js) ---
export const createPost = async (req, res) => {
  try {
    const { userId, description, picturePath } = req.body;
    const user = await User.findById(userId);
    const newPost = new Post({
      userId,
      firstName: user.firstName,
      lastName: user.lastName,
      location: user.location,
      description,
      userPicturePath: user.picturePath,
      picturePath,
      likes: {},
      comments: [],
    });
    await newPost.save();
    const post = await Post.find().sort({ createdAt: -1 });
    res.status(201).json(post);
  } catch (err) {
    res.status(409).json({ message: err.message });
  }
};
